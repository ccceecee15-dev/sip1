1. Screen intent (lock this)

This step allows a planner to:

Select a Store

View that store’s fixture-based capacity

See eligible styles for that store

Assign Facings and Depth per style

Validate usage against store capacity

This is a store-level physical feasibility screen.

2. Entry & empty state

Screen loads in an empty state

Show placeholder:

“Select a Merchandise Area and Store to define facings and presentation”

No data is shown until:

Merchandise Area is selected

Store filters are applied

3. Screen A — Store selection view (top-level)
Filters (top)

Merchandise Area (mandatory)

Format

Region

Required columns:

Store ID

Total Facings Capacity (read-only)

Total Presentation Capacity (read-only)

Status (e.g. OK / Over Capacity / Not Configured)

Action: View Store

No editing happens on this screen.

4. Drill-down behavior (required)

Clicking View Store navigates to Screen B

Context (Merch Area + Store) is carried forward

No modal — this should feel like a full screen

7. Live validation behavior

When the user edits:

Facings or Depth on any row

The system must:

Recalculate Presentation for that row

Update:

Facings used vs total

Presentation used vs total

Update status indicators immediately

No hard blocking required — warnings are sufficient.

8. What this screen must NOT do

❌ Do not allow editing fixture types

❌ Do not show SKU-level rows

❌ Do not allocate inventory

❌ Do not forecast demand

❌ Do not apply replenishment logic

This is physical shelf planning only.
