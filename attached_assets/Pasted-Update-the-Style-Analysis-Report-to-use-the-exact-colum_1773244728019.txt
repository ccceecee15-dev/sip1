# Table Behavior

Because the table contains many columns:

### Column Group Selector

Add a **Column Groups control above the table** where users can toggle visibility of each group.

Example:

Column Groups
☑ Product Attribute
☐ Logistics
☐ Sales Units
☐ Stock Units
☐ WOS
☐ Store Availability
☐ Promotion
☐ Orders
☐ Sales by Store

When a group is enabled, **all columns within that group should appear in the table**.

---

### Default View

Default visible groups:

* Product Attribute
* Sales Units
* Stock Units

All other groups should remain available but hidden.

---

### Horizontal Navigation

Because of the width of the table:

* Enable horizontal scrolling
* Freeze the first two columns:

  * Rank
  * Style Code

---

### Group Headers

The table should show **two header rows**:

Row 1 → Column Group Name
Row 2 → Individual Column Names

Example:

| Product Attribute | Product Attribute | Product Attribute | Sales Units | Sales Units |
| ----------------- | ----------------- | ----------------- | ----------- | ----------- |
| Rank              | Style Code        | Brand             | WTD         | LW          |

---

### Pagination

Add pagination at the bottom:

Rows per page:

* 25
* 50
* 100

Include Previous / Next navigation.

---

### Export

Export to Excel should include **all currently visible columns**, maintaining the column group structure.
